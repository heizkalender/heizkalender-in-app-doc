# Kalender-Ressourcen-ID 

Die Kalender-Ressourcen-ID ist die Identität der Ressource wie sie im Kalender definiert ist.

*Typische Werte sind: bei ChurchTools: „1“, bei ChurchDesk: „12348“; bei Google Kalender und bei Standard ICS (iCal): „#Saal“*

## Hinweise

So ermitteln Sie die IDs mit Ihrem Kalender-Dienst: 

### ChurchTools

Siehe Menü: Resources / Stammdaten / Ressource ➔ erste Ziffer der Zeile = Ressourcen-ID


### ChurchDesk

Siehe Menü: Nutzer / ChurchTools Einstellungen / Gemeinde & Ressourcen 

➔ aus der Tabelle eine Ressource auswählen ➔ Aktionen ➔ exportieren ➔ Öffentlicher Kalender ➔ aus dem Link die Zahl hinter „resource“ auswählen

*Beispiel: https://api2.churchdesk.com/ical/resource/95966?organizationId=7025 ➔ Ressourcen-ID = 95966
  

### Google-Kalender und Standard ICS (iCal)

Beim Google-Kalender bzw. ICS gibt es keine ID-Nummern im eigentlichen Sinne.

Stattdessen werden die vom Admin bestimmten Namen als Ressourcenidentität verwendet.

Ein solcher Name (z.B.: „#Saal“) muss als ein Textabschnitt im Titel des Termins enthalten sein.

Das Programm sucht im Titel des Termins nach diesem Namen. Er muss also irgendwo im Titel genauso vorkommen z.B.: „Treffen im #Saal zur Theaterprobe“. 

Ein Sonderzeichen zu nutzen ist nicht nötig, aber hilfreich, z.B. „Saal*“ oder „#Saal“.

- Ein Sonderzeichen kann verhindert, dass durch versehentliches Erwähnen eines solchen Bergriffes (z.B. Saal) ungewollt geheizt wird. 

- Man kann mit Anwendung eines Sonderzeichens einen Raum ohne oder mit Heizung reservieren, z.B.:

  - "Treffen im #Saal zur Theaterprobe“: Es **wird geheizt**, weil das Sonderzeichen dabei ist.
  - „Treffen im Saal zur Theaterprobe“. Es wird **nicht geheizt**, weil das Sonderzeichen fehlt.
  
**Wichtig: Verbotene Sonderzeichen sind *;* (Semikolon) und *$* (Dollar). Sie führen dazu, dass der Heizkalender nicht wie erwartet funktioniert.**
